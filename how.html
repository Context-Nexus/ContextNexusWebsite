How Context Nexus Works

Context Nexus is an in-editor AI co-pilot for Unreal Engine. It builds a structured model of your project—across C++ and Blueprints—so the AI can answer architectural questions accurately and cheaply.

1) Global Analysis (Deep Scan)

We parse the entire C++ target (headers + sources) and all Blueprint assets to produce a project knowledge graph. This graph contains types, functions, variables, call sites, event flows, and cross-references between C++ and Blueprints.

C++: Clang AST (Planned/Rolling Out)

We generate a compile_commands.json for the target and run a sidecar helper that uses Clang’s AST to parse your code with compiler-grade accuracy. This extracts class/struct definitions, UFUNCTION/UPROPERTY, function bodies, parameters, templates, inheritance, UE macros, RPC qualifiers, and symbol references.

Blueprint: Blueprint Bridge (Shipping)

Our system programmatically traverses Blueprint graphs (nodes, pins, edges). It resolves the Ubergraph to follow event stubs (BeginPlay, ReceiveTick, RPC events) to their real execution path and emits a compact, token-efficient intermediate representation (IR) that preserves logic and dependencies.

2) Targeted Context (Selective Send)

You choose the exact files, functions, variables, or graphs relevant to your question. Context Nexus builds a minimal context payload from the knowledge graph and sends only that to the AI, resulting in lower token costs and higher answer precision.

3) Answering Architectural Questions

This deep understanding allows you to ask complex queries like:

  • “If I change AMyActor::Tick, which Blueprints break?”
  • “Show everything that writes to PlayerHealth.”
  • “Which BP calls end up invoking StartMatch() on the GameMode?”

Under the hood, we perform a dependency traversal over the merged C++/BP graph, analyzing edge types like calls, read/writes, events, overrides, and delegate binds.

Why AST Beats Regex for AI Copilots in Unreal

TL;DR: Regex can match text; an Abstract Syntax Tree (AST) understands code. For Unreal’s macro-heavy, template-rich C++, the difference is night and day.

1) Unreal C++ Isn’t Regular

Unreal C++ makes heavy use of macros (UCLASS, UFUNCTION), templates (TArray, TMap), and complex qualifiers. Regex breaks on the first edge case; an AST is built to parse every case the compiler accepts.

2) AST Gives Ground Truth

An AST provides exact symbol tables, real cross-translation-unit reference mapping (not best-guess string matches), and accurate types after macro expansion.

3) Token Budgets Love Precision

An AST creates a precise dependency graph, which leads to a minimal context payload. Regex produces noisy, duplicated matches, resulting in bloated prompts, higher costs, and worse answers.

4) Safety & Isolation

Our AST parser runs in a sidecar process. A crash in parsing a malformed file can’t take down the editor. It also provides clear, actionable diagnostics for issues like bad includes or missing defines.

5) Better Cross-Language Answers

The AST’s truth for C++ combined with the Blueprint Bridge’s truth for Blueprints yields a unified, typed graph. That’s what enables questions like “Which Blueprints will break if I change this C++ function?”

Implementation Sketch (High-Level)

  1. Generate compile_commands.json from UBT for your target.
  2. Launch our sidecar helper with Clang libraries.
  3. Parse Translation Units and emit JSON with symbols, relations, and references.
  4. Merge with Blueprint Bridge output to create the unified graph.
  5. Build Targeted Context for the user’s query and send it to the AI.
